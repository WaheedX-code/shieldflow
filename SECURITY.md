# Security Policy

## Supported Versions

ShieldFlow is currently in active development. The following versions receive security updates:

| Version | Supported |
| ------- | --------- |
| latest (main) | ✅ |

## Reporting a Vulnerability

If you discover a security vulnerability in ShieldFlow, please **do not** open a public issue.

Instead, report it privately using one of these methods:

- **GitHub Private Vulnerability Reporting** — Use the Security tab in this repository
- **Email** — Reach out directly via GitHub profile: [WaheedX-code](https://github.com/WaheedX-code)

## What to Expect

- You will receive an acknowledgement within **48 hours**
- We will investigate and provide updates on the fix progress
- Once fixed, you will be credited in the release notes (unless you prefer anonymity)

## Scope

The following are considered in scope for vulnerability reports:
- Security misconfigurations in the pipeline workflow files
- Vulnerabilities that could expose developer secrets or scan results
- Logic flaws that could bypass security checks

Thank you for helping keep ShieldFlow and its users safe! 🛡️
